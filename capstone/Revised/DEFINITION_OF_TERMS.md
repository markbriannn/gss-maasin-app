# DEFINITION OF TERMS

This section provides operational definitions of key terms used throughout the study to ensure clarity and consistency in understanding the General Service System.

**Administrator**  
Refers to the authorized system user responsible for managing and overseeing the General Service System. The administrator performs critical functions including service provider verification, registration approval, booking request validation, system activity monitoring through real-time map view and analytics dashboard, transaction oversight, and resolution of user concerns to maintain platform integrity and security.

**Admin Approval Workflow**  
The systematic process wherein booking requests submitted by clients undergo administrative review and approval before being forwarded to service providers for acceptance or rejection. This workflow ensures quality control, payment verification, and compliance with platform policies.

**Barangay Clearance**  
An official document issued by the local barangay government, required by the system as part of the service provider verification process. This clearance confirms the provider's residency, good moral character, and standing within the local community, serving as a foundational document for identity verification.

**Booking**  
A formal service transaction initiated by a client through the system, containing detailed information about the requested service, scheduled date and time, service location, selected payment method, and agreed payment amount. The booking serves as a contractual agreement between the client and service provider.

**Booking Status**  
The current state of a service request within the system lifecycle, progressing through defined stages: Pending (awaiting admin approval) → Accepted (provider confirmed) → Traveling (provider en route) → Arrived (provider at location) → In Progress (service being performed) → Completed (service finished and confirmed).

**Brevo**  
A cloud-based email service platform integrated into the system to send verification codes and system notifications to users via email during registration and throughout platform usage.

**Client**  
A registered system user who utilizes the platform to search for, evaluate, request, and hire verified service providers for sideline or short-term services. Clients have access to features including provider browsing, booking creation, payment processing, job tracking, messaging, and feedback submission.

**Digital Receipt**  
An electronic transaction record automatically generated by the system upon service completion, documenting comprehensive details including service description, provider information, payment amounts, platform fees, timestamps, and completion status for record-keeping and accountability purposes.

**Email Verification**  
A security process implemented during user registration that requires users to confirm their email address by entering a verification code sent through the Brevo email service. This process ensures the validity of user contact information and prevents fraudulent account creation.

**Feedback and Rating System**  
A post-service evaluation feature that enables clients to assess service provider performance by submitting numerical ratings (1-5 stars) and written comments. This system promotes accountability, helps maintain service quality, influences provider rankings, and assists future clients in making informed decisions.

**Firebase**  
A comprehensive cloud-based platform by Google utilized in the system architecture for multiple functions: user authentication (Firebase Authentication), real-time database storage (Cloud Firestore), push notifications (Firebase Cloud Messaging), file storage (Firebase Storage), and web hosting (Firebase Hosting).

**Gamification**  
A system feature that incorporates game-like elements including points, levels, tiers (Bronze, Silver, Gold, Platinum), badges, and achievements to encourage user engagement and reward positive behaviors. The gamification system tracks user activities such as completed bookings, spending patterns, and platform participation.

**General Service System (GSS)**  
The web-based and mobile platform developed in this study that facilitates connections between clients and verified service providers in Maasin City, Southern Leyte. The system provides comprehensive features for service discovery, booking management, secure payments, real-time tracking, in-app messaging, and performance evaluation.

**Government-Issued ID**  
An official identification document issued by a recognized government agency (such as a driver's license, passport, national ID, voter's ID, or PhilHealth ID) required during service provider registration. This document serves as primary proof of identity and is used in conjunction with other verification requirements.

**GPS Tracking**  
A system feature utilizing Global Positioning System technology to display the real-time geographical location of service providers on an interactive map interface. This functionality allows clients to monitor provider movement, estimate arrival times, and track service progress during active job execution.

**Identity Verification**  
A comprehensive multi-step authentication process wherein service providers submit required documents including a government-issued ID, profile photograph, Barangay Clearance, and Police Clearance. This rigorous verification ensures user authenticity, enhances platform security, and builds trust within the community.

**In-App Messaging**  
A built-in real-time communication feature that enables clients, service providers, and administrators to exchange text messages and images directly within the platform for service coordination, clarification, and support without requiring external communication tools.

**Job Request**  
A formal service inquiry submitted by a client through the system interface, containing specific details about the required service type, preferred schedule, service location, selected payment method, additional requirements, and supporting photographs or documentation. The job request initiates the service matching and booking process.

**Job Tracking**  
A comprehensive real-time monitoring feature that enables both clients and service providers to track the current status, location, and progress of an active service engagement. This feature provides transparency throughout the service lifecycle, from provider acceptance through travel, arrival, service execution, and completion.

**Leaderboard**  
A gamification feature that displays ranked lists of top-performing clients and service providers based on accumulated points, completed transactions, and platform engagement. The leaderboard includes percentile rankings and tier classifications to encourage healthy competition.

**Mobile Application**  
The React Native-based version of the General Service System designed for Android and iOS devices, providing native mobile experience with full system functionality including offline capabilities, push notifications, and device-specific features.

**Next.js**  
A React-based web framework utilized to develop the web application version of the General Service System, providing server-side rendering, optimized performance, and seamless user experience across desktop and mobile browsers.

**Online Payment**  
A secure digital payment mechanism integrated into the system that enables clients to pay for services through the PayMongo payment gateway, supporting GCash e-wallet and credit/debit card transactions. This feature ensures transparent, traceable, and properly documented financial transactions while protecting both parties.

**PayMongo**  
A Philippine-based payment gateway service integrated into the system to process online payments securely. PayMongo supports multiple payment methods including GCash e-wallet and major credit/debit cards, ensuring compliance with local payment regulations.

**Platform Service Fee**  
A 5% fee added to the client's total payment as system revenue, ensuring service providers receive their full quoted price while supporting ongoing platform maintenance, operational costs, infrastructure expenses, and continuous improvements.

**Police Clearance**  
An official document issued by the Philippine National Police (PNP) certifying that the service provider has no criminal record or pending criminal cases. This clearance is a mandatory requirement in the verification process to ensure user safety and maintain platform credibility.

**Push Notification**  
Real-time alerts delivered to users' devices through Firebase Cloud Messaging (FCM) to inform them of critical events including booking updates, admin approvals, provider acceptance, new messages, provider arrival notifications, payment confirmations, and other system events requiring immediate attention.

**React Native**  
A JavaScript framework developed by Meta (Facebook) used to develop the mobile application version of the General Service System, enabling cross-platform development for both Android and iOS devices from a single codebase while maintaining native performance.

**Real-Time Location Tracking**  
A GPS-based feature that continuously monitors and displays the service provider's current geographical location during active job execution. This functionality enhances transparency, allows clients to estimate arrival times, and provides an additional layer of security for both parties.

**Semaphore**  
An SMS gateway service integrated into the system to send One-Time Password (OTP) verification codes to users' mobile phones during registration and authentication processes, ensuring valid phone number verification.

**Service Categories**  
A classification system that organizes available services into distinct groups such as Electrical, Plumbing, Carpentry, Cleaning, Transportation, Appliance Repair, and other specialized trades. This categorization facilitates efficient service discovery and provider-client matching.

**Service Listing**  
A comprehensive display feature that presents verified service provider information including offered services, professional skills, years of experience, pricing structure (fixed or hourly), availability status, average ratings, review count, completed jobs, and profile details. This listing enables clients to make informed hiring decisions.

**Service Provider**  
A registered, verified, and approved individual who offers sideline services through the platform. Service providers undergo rigorous identity verification, maintain active profiles showcasing their skills and experience, and deliver services such as electrical work, plumbing, carpentry, cleaning, transportation, and other specialized trades.

**Sideline Services**  
Short-term, part-time, or project-based services offered by individuals to supplement their primary income. These services typically include skilled trades, manual labor, transportation, and other specialized tasks that do not require full-time employment commitments.

**SMS Verification**  
A security authentication process implemented during user registration that requires users to verify their mobile phone number by entering a One-Time Password (OTP) sent via SMS through the Semaphore service. This verification ensures valid contact information and adds an additional security layer.

**System Verification**  
The comprehensive administrative process of reviewing, validating, and approving service provider accounts based on submitted documentation and compliance with system requirements. This process includes document authentication, background verification, and final approval or rejection by the system administrator.

**Transaction History**  
A chronological record of all completed bookings, payments, and service engagements maintained within the system. This feature provides users with access to past activities, digital receipts, and documentation for reference, dispute resolution, and accountability purposes.

**User Account**  
A registered profile within the system that grants access to role-specific features and functionalities. User accounts are categorized into three types: Client, Service Provider, and Administrator, each with distinct permissions and capabilities aligned with their respective roles. Accounts are created through email and phone verification processes.

**Web Application**  
The Next.js-based version of the General Service System accessible through internet browsers on desktop computers, laptops, tablets, and mobile devices without requiring local software installation. The web application provides full system functionality with responsive design.

**Web-Based Platform**  
A software application architecture that operates through internet browsers on various devices without requiring local software installation. This architecture ensures cross-platform compatibility, easy accessibility, automatic updates, and centralized data management.
