## Process Descriptions

| Process | Name | Description |
|---------|------|-------------|
| **1** | Client Registration | User registers as client with personal info, email/phone verification, location |
| **2** | Provider Registration | Service provider registers with documents (ID, clearance), service category, location, profile photo |
| **3** | Job Matching | Client browses online providers on map, filters by category, sorts by recommended/cheapest/nearest |
| **4** | Job Process | Client creates booking with photos/videos of problem, sets address, selects payment method |
| **5** | Provider Approval | Admin verifies documents and approves/rejects/suspends provider applications |
| **6** | Payment Processing | GCash/Maya payment via PayMongo (Pay First model), handles refunds |
| **7** | Admin Review | Admin reviews booking request, approves or rejects (auto-refund if rejected) |
| **8** | Provider Response | Provider receives notification, accepts or declines the job |
| **9** | Service Execution | Provider travels (real-time tracking), arrives, starts work, completes service |
| **10** | Feedback & Rating | Client confirms completion, rates provider, payment released to provider wallet (95%) |

| **Favorites** | Firestore collection for saved providers | userId, providerId |
| **Messages** | Firestore collection for chat | conversationId, senderId, receiverId, text |
